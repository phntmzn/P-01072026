# 08-01072026
Here’s **`grep` octal searching** — how to search for **non-printable characters** (NULs, control chars, raw bytes) using **octal escape sequences**.

---

## What “octal searching” means
`grep` lets you specify characters by **octal byte value** using:

```
\ooo   (ooo = 1–3 octal digits, 0–7)
```

This is mainly used to match:
- control characters
- binary data
- protocol delimiters
- non-printable bytes

---

## Basic syntax
```bash
grep $'pattern\ooo' file
```

⚠️ **Important**:  
You must use **ANSI-C quoting** (`$'...'`) so the shell interprets `\ooo`.

---

## Common octal values

| Octal | Hex | Character |
|-----|----|---------|
| `\000` | 0x00 | NUL |
| `\011` | 0x09 | TAB |
| `\012` | 0x0A | LF (newline) |
| `\015` | 0x0D | CR |
| `\033` | 0x1B | ESC |
| `\040` | 0x20 | Space |
| `\177` | 0x7F | DEL |

---

## Example: search for a TAB character
```bash
grep $'\011' file.txt
```

Equivalent to:
```bash
grep $'\t' file.txt
```

---

## Example: search for NUL bytes (binary files)
```bash
grep -a $'\000' binaryfile
```

- `-a` → treat binary as text

---

## Example: search for ESC (often in terminal control sequences)
```bash
grep -a $'\033\[' file.bin
```

Matches ANSI escape sequences like `ESC[`.

---

## Octal vs hex escapes
`grep` itself supports **octal**, not `\xNN` hex escapes.

❌ This will NOT work:
```bash
grep "\x1b" file
```

✔ Correct:
```bash
grep $'\033' file
```

---

## Combining octal with text
```bash
grep -a $'ERROR\015\012' file.bin
```

Matches `ERROR\r\n` (Windows line endings).

---

## Octal ranges (advanced)
To match *any control character*:
```bash
grep -a $'[\000-\037]' file.bin
```

To find **non-ASCII bytes**:
```bash
grep -a $'[\200-\377]' file.bin
```

---

## Fixed-string + octal (literal bytes)
```bash
grep -a -F $'\000PNG' file.bin
```

Matches a PNG header after a NUL.

---

## Practical use cases
- Detect **binary blobs in logs**
- Find **embedded NULs** in text files
- Analyze **malware / packed files**
- Inspect **network captures**
- Hunt **protocol delimiters**

---

## Quick sanity check (generate test data)
```bash
printf 'A\011B\n' > test.txt
grep $'\011' test.txt
```

