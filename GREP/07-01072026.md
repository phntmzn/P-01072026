# 07-01072026
Here’s **`grep` fixed-string matching** — when you want **literal text only**, no regex behavior, maximum safety and speed.

---

## What “fixed strings” means
With **`-F`**, `grep` treats the pattern as **plain text**:
- ❌ no regex
- ❌ no metacharacters
- ✔ exact byte/string match
- ✔ faster than regex

```bash
grep -F "text" file.txt
```

---

## Why use `-F`
Use fixed strings when:
- searching for **log messages**
- searching for **code literals**
- searching for **paths, URLs, hashes**
- avoiding regex escaping hell
- performance matters

---

## Basic fixed-string search
```bash
grep -F "ERROR [code=42]" app.log
```

(No escaping required)

---

## Case-insensitive fixed match
```bash
grep -Fi "connection reset" app.log
```

---

## Multiple fixed strings (OR)

### Using multiple `-e`
```bash
grep -F -e "ERROR" -e "FATAL" app.log
```

### From a pattern file
```bash
grep -F -f patterns.txt app.log
```

`patterns.txt`:
```
ERROR
FATAL
panic occurred
```

---

## Recursive fixed-string search
```bash
grep -rF "API_KEY=" .
```

---

## Output control works normally
```bash
grep -F -n "timeout" app.log
grep -F -l "TODO" -r .
grep -F -C 3 "CRITICAL FAILURE" app.log
```

---

## Fixed strings vs regex (quick contrast)

| Task | Regex | Fixed (`-F`) |
|---|---|---|
| Match literal `.` | escape `\.` | just `.` |
| Match `[ERROR]` | escape `\[ERROR\]` | `[ERROR]` |
| Speed | slower | **faster** |
| Safety | pattern-sensitive | **safe** |

---

## Binary & large files
```bash
grep -F -a "PNG" image.bin
```

---

## Quiet / script usage
```bash
grep -F -q "READY" app.log && echo "OK"
```

---

## Fixed string gotchas
- No partial logic (use multiple `-e`)
- No anchors (`^`, `$`)
- No character classes

If you need *any* regex → **don’t use `-F`**

---

## Ultra-common real-world patterns

### Find exact log lines
```bash
grep -F "Server started on port 8080" app.log
```

---

### Find hard-coded secrets
```bash
grep -rF "AKIA" .
```

---

### Find exact paths
```bash
grep -F "/usr/local/bin/python" file.txt
```

---

## Performance tip (important)
If your pattern **does not need regex**, always prefer:
```bash
grep -F
```
or even better:
```bash
rg -F
```

---

## Mental model
```
-F  → literal bytes only
-e  → multiple literals
-f  → list of literals
```
